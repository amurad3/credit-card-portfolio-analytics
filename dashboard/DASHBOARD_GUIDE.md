# Cardholder Risk & Delinquency Dashboard — Build Guide

This guide documents the interactive **cardholder risk dashboard** built on the
extracts in [`dashboard/extracts/`](extracts). The extracts are plain CSVs, so
the same workbook can be reproduced in **Tableau Public**, **Power BI Desktop**,
or Excel. Static reference renders of the key views live in
[`dashboard/charts/`](charts).

## Data sources (extracts)

| File | Grain | Used for |
| --- | --- | --- |
| `risk_tier_kpis.csv` | one row per risk tier | KPI tiles, tier scorecard, risk-concentration bars |
| `kmeans_segments.csv` | one row per KMeans segment | unsupervised-segment comparison |
| `delinquency_by_utilization.csv` | one row per utilization band | "delinquency vs. utilization" driver chart |
| `delinquency_by_filedepth.csv` | thin vs. established file | thin-file driver chart |
| `delinquency_interaction.csv` | utilization × file-depth | interaction heatmap |
| `delinquency_drivers.csv` | one row per driver | logistic-regression odds-ratio bars |
| `monthly_delinquency_trend.csv` | one row per month | delinquency / charge-off trend lines |
| `account_scored_sample.csv` | 8,000 sampled accounts | utilization-vs-FICO scatter, drilldowns |

All extracts are regenerated by `python/analytics.py`, which pulls from the
cleaned tables in `data/processed/`. The metric definitions match the SQL in
`sql/`, so the BI layer, the SQL layer, and the Python layer report identical
numbers.

## Dashboard layout (4 sheets + 1 dashboard)

**Sheet 1 — Executive KPI band**
Big-number tiles: total accounts, outstanding balance, 30+ delinquency rate,
charge-off rate, average utilization. Portfolio values are in
`reports/kpis.json`.

**Sheet 2 — Risk-tier scorecard**
Source: `risk_tier_kpis.csv`. A table/bar of the four tiers with delinquency
rate, charge-off rate, average FICO, average utilization, and share of
charged-off balance. Color tiers green → red (Tier 1 → Tier 4). This is the
segmentation centerpiece and shows risk concentration: Tiers 3–4 are a small
slice of accounts but the majority of charge-off balance.
(Static render: `charts/risk_tier_kpis.png`.)

**Sheet 3 — Root-cause drivers**
Two views side by side:
- `delinquency_by_utilization.csv` → delinquency rate rising across utilization
  bands (`charts/delinquency_by_utilization.png`).
- `delinquency_drivers.csv` → logistic-regression odds ratios per +1 SD, with a
  reference line at 1.0 (`charts/delinquency_drivers.png`).
Optionally add `delinquency_interaction.csv` as a 2×2 highlight table of
utilization × file depth.

**Sheet 4 — Trend & vintage**
Source: `monthly_delinquency_trend.csv`. Dual-line chart of 30+ delinquency and
charge-off rate over the window, showing the rising trend
(`charts/delinquency_trend.png`).

**Dashboard — "Cardholder Risk Overview"**
Combine the four sheets. Add a **risk-tier filter**, a **card-product filter**,
and a **region filter** (all available from the extracts) and set them to apply
across the dashboard. Add a **utilization-band highlighter** so a reviewer can
isolate high-utilization accounts instantly.

## Recreating in Tableau

1. *Connect → Text file* → select the `extracts/` folder.
2. Build sheets 1–4 as above.
3. Assemble the dashboard, add the tier / product / region filters, and set them
   as global (Dashboard → Actions → Apply to all using this data source).
4. Publish to Tableau Public and link it from the project README.

## Recreating in Power BI

1. *Get Data → Folder* → point at `extracts/`, then expand each CSV as its own
   query.
2. Recreate the four pages; use slicers for risk tier, product, and region.

The risk-tier definition and all metric thresholds match
`sql/04_risk_segmentation.sql` and `python/analytics.py`, so every layer reports
the same figures.
